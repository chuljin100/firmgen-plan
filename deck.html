Problem

펌웨어 개발의 구조적 문제 3가지

국내 제조사 임베디드 현장 공통 — 현장 인터뷰에서 수집.

1

FW 개발자를 못 뽑는다

임베디드 채용난 심화. HW 엔지니어는 있지만 펌웨어 짤 사람이 없다.

"채용 공고 6개월째 비어있어요."
2

그 한 명이 나가면 끝

FW는 1인 개발 관행. 담당자 퇴사 = 프로젝트 드랍 또는 수정 불능 블랙박스.

"코드는 있는데 빌드가 안 돼요."
3

FW가 일정의 병목

핀맵 협의부터 브링업까지 HW가 끝나도 FW를 기다리며 제품 전체가 멈춘다.

"보드 나온 지 두 달, 아직 LED도 못 켰어요."

Why Now

클라우드 AI가 들어갈 수 없는 시장

코드 생성 AI는 웹 개발의 표준이 됐지만, 임베디드는 사각지대다.

보안 규정

제조사 펌웨어는 소스 외부 전송 금지가 다수. Copilot을 쓰고 싶어도 못 쓴다. 온프레미스가 유일한 답.

도메인 갭

범용 LLM은 HAL 버전 혼동, 없는 레지스터 생성, 핀맵 오류. 데이터시트 근거 RAG 없이는 실무 투입 불가.

오픈모델 성숙

30B~200B급 오픈 코더 모델이 상용에 근접. 사내 GPU 서버로 충분한 품질 시점 도래.

Key Features

네 가지 차별화 — 경쟁자가 못 베끼는 부분

핀 어시스턴트

HW·FW가 회의로 하던 핀 협의를 시스템이 대신. HW 고정핀 제약 입력 → 충돌 검사 → 핀맵 확정 즉시 FW 뼈대 생성. HW 엔지니어가 FW 없이 브링업까지.

검증 파이프라인

생성 코드를 그대로 내보내지 않는다. 게이트 전부 통과한 프로젝트만 제공.

arm-gccMISRA-C유닛테스트Renode 시뮬HIL

표준 산출물 패키지

모든 프로젝트가 국제 SW 밸리데이션 표준(IEC 62304 · ISO 14971 · IEEE 830/1012) 문서 체계로 나온다. SRS·설계서·시험기록·추적성 매트릭스를 파이프라인이 자동 생성 — 사후 문서화 제로. 코드가 아니라 감사 가능한 자산을 배달한다. 인증 필요한 고객에겐 인허가 기술문서 초안이 공짜로 따라온다.

지식 자산화

프로젝트 = 명세+설계결정+핀맵+테스트 저장소. 담당자가 없어도 재생성·수정 가능. "퇴사 보험" — 그 자체로 영업 포인트이자 구독 유지 논리.

Risk & Mitigation

품질 격차

데모 vs 온프렘 클레임

격차를 티어표에 명시. 계약 전 고객 프롬프트로 온프렘 모델 실측 데모.

경쟁

오픈모델 발전 → 자체 구축 시도

해자는 모델이 아니라 RAG 인덱스·검증 루프·유지보수. 구독으로 락인.

원가

GPU 시세 변동

견적 유효기간 30일. HW는 발주 시점 시세 반영 조항.

책임

생성 코드 결함 사고

"개발 보조 도구, 최종 검증 책임은 고객" 계약 명시. 기능안전 용도 제외.
