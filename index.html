The Problem

펌웨어 개발, 이 세 가지가 발목 잡고 있지 않습니까

국내 제조사 임베디드 현장에서 반복되는 구조적 문제입니다.

FW 개발자를 못 뽑는다

임베디드 개발자 채용은 해가 갈수록 어려워지고, 연봉은 올라갑니다. 채용 공고 6개월째 비어있는 자리, 익숙하지 않으십니까.

"HW 엔지니어는 있는데 FW를 짤 사람이 없어요."

그 한 명이 나가면 끝

펌웨어는 1인 개발이 관행입니다. 담당자가 퇴사하면 프로젝트가 드랍되거나, 아무도 못 고치는 블랙박스가 됩니다.

"코드는 있는데 빌드가 안 돼요. 문서요? 없죠."

HW는 끝났는데 FW를 기다린다

회로는 완성됐지만 펌웨어가 안 나와서 제품 전체가 멈춥니다. 핀맵 협의부터 브링업까지, FW가 항상 일정의 병목입니다.

"보드는 두 달 전에 나왔는데 아직 LED도 못 켰습니다."
Who It's For

당신이 누구든, 펌웨어를 만들 수 있게

FirmGen은 코드 자동완성 도구가 아닙니다. 요구사항에서 검증된 프로젝트까지, 개발 과정 자체를 시스템이 수행합니다.

How It Works

"컴파일 되는 코드"가 아니라 "동작이 검증된 펌웨어"

생성형 AI의 그럴듯한 코드를 그대로 내보내지 않습니다. 5단계 파이프라인의 검증 게이트를 전부 통과한 프로젝트만 손에 쥐어드립니다.

요구사항 명세

자연어로 쓰면 시스템이 되물어 명세 완성. 명세가 프로젝트의 원본.

핀·주변장치 설계

데이터시트 AF표 기반 핀 할당, HW 고정핀 제약 반영, CubeMX 호환 출력.

프로젝트 생성

스니펫이 아닌 빌드 가능한 전체 프로젝트 — 드라이버·태스크·빌드 스크립트.

다층 검증

컴파일 → MISRA-C 정적분석 → 유닛테스트 → 시뮬레이션 동작 검증.
